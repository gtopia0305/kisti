**6. module 사용법 (애플리케이션 사용을 위한 환경 설정 툴)**&#x20;

뉴론 시스템에 설치된 컴파일러, 라이브러리, 주요 애플리케이션에 대해 module을 활용하여 쉽게 환경설정을 할수 있도록 하였습니다.&#x20;

&#x20;

사용가능한 모듈 목록은 다음과 같습니다. (2021년 4월 현재)

| <p> <strong>$ module av</strong>  <br><br><br>--------------------------------------- /apps/Modules/modulefiles/compilers ---------------------------------------<br>gcc/4.8.5    gcc/8.3.0    intel/18.0.2 pgi/19.1<br><br>--------------------------------------- /apps/Modules/modulefiles/libraries ---------------------------------------<br>hdf4/4.2.13  hdf5/1.10.2  lapack/3.7.0 ncl/6.5.0    netcdf/4.6.1<br><br>------------------------------------------ /apps/Modules/modulefiles/mpi ------------------------------------------<br>cudampi/mvapich2-2.3  cudampi/openmpi-3.1.0  cudampi/openmpi-3.1.5  mpi/impi-18.0.2       <br>mpi/mvapich2-2.3      mpi/openmpi-3.1.0     mpi/openmpi-3.1.5<br><br>---------------------------------- /apps/Modules/modulefiles/libraries_using_mpi ----------------------------------<br>fftw_mpi/2.1.5 fftw_mpi/3.3.7<br><br>------------------------------------- /apps/Modules/modulefiles/applications --------------------------------------<br>cmake/3.12.3        gaussian/g16.b01    htop/3.0.5          namd/2.12           python/3.7.1        R/3.5.0<br>cuda/10.0           gaussian/g16.c01    java/openjdk-11.0.1 nvtop/1.1.0         qe/6.4.1_k40        singularity/3.1.0<br>gaussian/g16        gromacs/2016.4      lammps/16Mar18      python/2.7.15       qe/6.4.1_v100       singularity/3.6.4<br><br>------------------------------------ /apps/Modules/modulefiles/conda_packages -------------------------------------<br>conda/caffe_gpu_1.0   conda/pytorch_1.0     conda/tensorflow_1.13<br><br><br></p> |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

&#x20;

\[module av 목록의 주요 카테고리 설명]

\>> /apps/Modules/modulefiles/applications :  주요 애플리케이션들의 모음&#x20;

\>> /apps/Modules/modulefiles/compilers : 사용 가능한 컴파일러들의 모음&#x20;

\>> /apps/Modules/modulefiles/libraries : 주요 라이브러리들의 모음

\>> /apps/Modules/modulefiles/mpi :  사용 가능한 MPI 들의 모음(cudampi 는 cuda sdk에 의존성이 있는 MPI)

\>> /apps/Modules/modulefiles/libraries\_using\_mpi : MPI 의존 라이브러리들의 모음

&#x20;

각 모듈들의 사용방법은 **module help** 명령을 사용하시면 간단한 사용 예제를 보실 수 있습니다.&#x20;

| <p> <strong>$ module help intel/18.0.2</strong> <br> <br>----------- Module Specific Help for 'intel/18.0.2' ---------------<br> <br>This module is for use of Intel Compiler 2018.<br>It needs module(s):<br>                  None<br>Use example:<br>        $ module load intel/18.0.2<br><br><br>Additional info:<br>    1. We can use Intel Advisor, Vtune, Inspector, TBB and MKL.<br>    2. Major environment variables is set up like these:<br>          CC=icc   CXX=icpc   FC=ifort    F77=ifort    F90=ifort</p> |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

&#x20;

module help 결과를 참고하여 다음과 같이 module load 명령어를 이용하여 intel-2018 컴파일러 환경을 설정합니다.&#x20;

|  **$ module load intel/18.0.2**  |
| -------------------------------- |

&#x20;

현재 모듈을 적재한 상황은 module list를 이용하여 확인이 가능합니다.&#x20;

| <p> <strong>$ module list</strong><br> Currently Loaded Modulefiles:<br>  1) intel/18.0.2</p> |
| --------------------------------------------------------------------------------------------- |

&#x20;

또한, 아래와 같은 방법으로 모듈에 적재한 애플리케이션 명령어가 PATH 환경 변수에 제대로 설정되어 있는지 확인합니다.

| <p> <strong>$ which icc</strong><br>/apps/compiler/intel/18.0.2/bin/icc</p> |
| --------------------------------------------------------------------------- |

&#x20;

추가로 cuda/10.0 툴킷 환경을 적재하고 다시 한번 module list로 확인합니다.

| <p> <strong>$ module load cuda/10.0</strong> <br> <strong>$ module list</strong><br>Currently Loaded Modulefiles:<br>  1) intel/18.0.2   2) cuda/10.0</p> |
| --------------------------------------------------------------------------------------------------------------------------------------------------------- |

&#x20;

&#x20;이번에는 더 이상 필요하지 않는 module 환경을 해제하도록 하겠습니다. 아래의 예에서는 cuda/10.0의 환경설정을 삭제하고 그 결과를 module list로 확인합니다.&#x20;

| <p> <strong>$ module rm cuda/10.0</strong> <br> <strong>$ module list</strong><br>Currently Loaded Modulefiles:<br>  1) intel/18.0.2  </p> |
| ------------------------------------------------------------------------------------------------------------------------------------------ |

&#x20;

혹은 module purge 명령어로 적재되었던 모든 module 들을 삭제해 버릴수도 있습니다.&#x20;

| <p> <strong>$ module purge</strong> <br> <strong>$ module list</strong><br>No Modulefiles Currently Loaded.<br> </p> |
| -------------------------------------------------------------------------------------------------------------------- |
